Panel chaired by Bill Gropp
Experts: Maria Garzaran, Stefano Markidis, Ron Brightwell

Designing MPI in 2020

MPI was designed over 25 years ago. In that era, computers required multiple chips for each processor; the ratio of memory latency to processor clock speed was far different, as was the ratio of floating point performance to memory and interconnect bandwidth. There has also been a tremendous change in how software is designed and used, with backward compatibility no longer required in many cases, and the lifetime of much (but not all) software now measured in months or years instead of decades. What would MPI look like if it were designed today? This panel will discuss some of the trends of current parallel computer architecture and software and how they might shape the design of a parallel programming system.
